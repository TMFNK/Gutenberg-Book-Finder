Every book card shows its cover, an LLM-generated hook, and tags, with a detail view linking straight to the free book on [gutenberg.org](https://gutenberg.org).

**M1**: Currently covering the **1,000 most-downloaded books**. Scaling to the full ~78,000‑book catalog is planned.

---

## ✨ Features

- **🔎 Full-text search**: Find any book by title, author, subject, or description instantly via [MiniSearch](https://github.com/lucaong/minisearch).
- **🎭 Filter by vibe**: Mood, theme, difficulty, subject, language, era; facets that actually help you discover.
- **🎲 Surprise me**: One click to a random book you might never have found.
- **🤖 LLM-enriched**: Each book gets a one-line hook, mood, theme tags, and difficulty rating, generated by an LLM (via [OpenRouter](https://openrouter.ai/)).
- **🗺️ 2D map**: Books are projected into a visual 2D space. Similar books cluster together, and you can explore by proximity.
- **📱 No backend**: Fully static site, deploys to GitHub Pages. No server to run.
- **🔁 Resumable pipeline**: Every stage caches its output. Re‑run or resume after any interruption.

## How it works

A Python pipeline builds the map data once, offline:

```mermaid
flowchart LR
  A[Catalog] --> B[Excerpts]
  B --> C[Embed]
  C --> D[Layout]
  D --> E[Enrich]
  E --> F[Export]
  F --> G[🧩 Frontend]
```

| Step         | Module                                                     | What it does                                                                                                                                |
| ------------ | ---------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Catalog**  | [`catalog.py`](pipeline/src/gutenberg_galaxy/catalog.py)   | Fetch book metadata (title, author, subjects, download counts) from the [Gutendex](https://gutendex.com/) API                               |
| **Excerpts** | [`excerpts.py`](pipeline/src/gutenberg_galaxy/excerpts.py) | Download each book's plain text, strip Gutenberg boilerplate, keep the first ~2,000 words                                                   |
| **Embed**    | [`embed.py`](pipeline/src/gutenberg_galaxy/embed.py)       | Embed title + subjects + excerpt locally with `sentence-transformers`: runs on-device, no API cost                                          |
| **Layout**   | [`layout.py`](pipeline/src/gutenberg_galaxy/layout.py)     | Reduce embeddings to 2D with UMAP, group into clusters with HDBSCAN                                                                         |
| **Enrich**   | [`enrich.py`](pipeline/src/gutenberg_galaxy/enrich.py)     | An LLM (via [OpenRouter](https://openrouter.ai/)) names each cluster and tags every book with mood, themes, difficulty, and a one-line hook |
| **Export**   | [`export.py`](pipeline/src/gutenberg_galaxy/export.py)     | Write compact JSON consumed by the frontend                                                                                                 |

The scraped metadata (catalog, LLM tags, exported book data) is committed to this repo, so the frontend runs **without re-running the pipeline**.

The frontend is a **static site**: Vite + TypeScript, no framework. Search is client‑side via [MiniSearch](https://github.com/lucaong/minisearch). No server required; deploys to GitHub Pages on every push to `main`.

## Project layout

```text

├── pyproject.toml
├── src/gutenberg_galaxy/

│   └── ...
└── public/data/             Pre-built JSON (committed)

```

## Running the pipeline

```bash
cd pipeline
uv sync
export OPENROUTER_API_KEY=...   # required for the enrich stage
uv run python -m gutenberg_galaxy all       # runs every stage
uv run python -m gutenberg_galaxy catalog   # or run one stage at a time
uv run pytest                                # run tests
```

Each stage caches its output under `data/` and skips work already done, so the pipeline is safe to re‑run or resume after an interruption.

---

## Running the frontend

```bash
cd web
npm install
npm run dev
```

Requires `web/public/data/books.json`, produced by the `export` pipeline stage (already committed to the repo).
